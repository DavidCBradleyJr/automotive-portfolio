  <action>
Create `scripts/migrate-to-cloudinary.js` as an ESM module (package.json has "type": "module").

The script must:

1. **Import and configure Cloudinary:**
```javascript
import { v2 as cloudinary } from 'cloudinary';
```
Configure with `process.env.CLOUDINARY_CLOUD_NAME`, `process.env.CLOUDINARY_API_KEY`, `process.env.CLOUDINARY_API_SECRET`.

2. **Fail fast on missing env vars:** Check all three env vars at startup. If any missing, print a clear error message and exit with code 1.

3. **Copy the exact categoryAssignments from process-images.js** (all 29 entries across 5 categories: jdm=6, euro=6, supercar=6, american-muscle=6, track=5).

4. **Upload each image sequentially** (to avoid rate limits on free tier):
```javascript
await cloudinary.uploader.upload(`public/images/gallery/${id}.webp`, {
  public_id: id,
  folder: `gallery/${category}`,
  context: { caption: caption, alt: altText },
  overwrite: true,
  resource_type: 'image',
});
```

5. **Log progress:** Print each upload as it completes: `[N/29] Uploaded {id} -> gallery/{category}/`

6. **Print summary:** Total uploaded count, any failures.

7. **After all uploads, verify by running a search query:**
```javascript
const result = await cloudinary.search
  .expression('folder:gallery/*')
  .max_results(100)
  .execute();
console.log(`\nVerification: ${result.total_count} images found in Cloudinary`);
```

8. **Add a "migrate" script to package.json:** `"migrate": "node --env-file=.env scripts/migrate-to-cloudinary.js"` (uses Node 20+ --env-file flag).

IMPORTANT: Use the object form for context metadata `{ caption, alt }` to avoid pipe/equals delimiter issues (per research pitfall 3).
  </action>
  <verify>
    <automated>node -c scripts/migrate-to-cloudinary.js && grep -q '"migrate"' package.json && echo "PASS"</automated>
  </verify>
  <acceptance_criteria>
