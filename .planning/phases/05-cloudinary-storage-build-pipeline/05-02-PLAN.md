Create `scripts/build-gallery-data.js` as an ESM module. This script runs during `npm run prebuild` and generates `src/data/gallery-images.js`.

**Structure:**

1. **Imports:**
```javascript
import { v2 as cloudinary } from 'cloudinary';
import { writeFile, mkdir } from 'node:fs/promises';
```

2. **Env var validation:** Check `CLOUDINARY_CLOUD_NAME`, `CLOUDINARY_API_KEY`, `CLOUDINARY_API_SECRET` exist. If any missing, print error with instructions to set them in Netlify dashboard or local .env, then `process.exit(1)`.

3. **Configure Cloudinary:**
```javascript
cloudinary.config({
  cloud_name: process.env.CLOUDINARY_CLOUD_NAME,
  api_key: process.env.CLOUDINARY_API_KEY,
  api_secret: process.env.CLOUDINARY_API_SECRET,
});
```

4. **Fetch all gallery images:**
```javascript
const result = await cloudinary.search
  .expression('folder:gallery/*')
  .with_field('context')
  .sort_by('public_id', 'asc')
  .max_results(500)
  .execute();
```

5. **Category label map** (hardcoded, same as v1):
```javascript
const categoryLabels = {
  'jdm': 'JDM',
  'euro': 'Euro',
  'supercar': 'Supercar',
  'american-muscle': 'American Muscle',
  'track': 'Track/Motorsport',
};
```

6. **Generate LQIP base64 with concurrency limit of 5:**
For each resource, build a tiny blurred thumbnail URL:
```javascript
const lqipUrl = cloudinary.url(resource.public_id, {
  width: 16,
  effect: 'blur:1000',
  quality: 'auto',
  fetch_format: 'webp',
  secure: true,
});
```
Fetch in batches of 5 using Promise.all, convert each response buffer to base64 data URL: `data:image/webp;base64,${buffer.toString('base64')}`.

7. **Build gallery entries array:**
For each resource from search results:
- `id`: Extract from `public_id` -- the filename part after the last `/` (e.g., `gallery/jdm/jdm-street-racer-01` -> `jdm-street-racer-01`)
- `src`: Build Cloudinary delivery URL using `cloudinary.url(resource.public_id, { fetch_format: 'auto', quality: 'auto', width: 2000, crop: 'limit', secure: true })`
- `lqip`: The base64 string generated above
- `width`: `resource.width`
- `height`: `resource.height`
- `category`: Extract from folder path -- the segment after `gallery/` (e.g., `gallery/jdm` -> `jdm`). Use `resource.asset_folder` if available (dynamic folder mode), otherwise parse from `resource.folder` or `resource.public_id`.
- `caption`: `resource.context?.custom?.caption || ''`
- `alt`: `resource.context?.custom?.alt || ''`
- `isPlaceholder`: Always `false`

8. **Sort entries:** By category alphabetically, then by id alphabetically (same as v1).

9. **Write the data file** to `src/data/gallery-images.js` with this EXACT format:
```
/**
 * Gallery Image Data
 *
 * Auto-generated by scripts/build-gallery-data.js
 * Do not edit manually -- re-run the build to regenerate.
 *
 * Total: {N} images
 */

export const galleryImages = [
  {
    id: '{id}',
    src: '{cloudinary_url}',
    lqip: '{base64_string}',
    width: {width},
    height: {height},
    category: '{category}',
    caption: '{caption}',
    alt: '{alt}',
    isPlaceholder: false,
  },
  // ...
];

export const categories = [
  { id: 'all', label: 'All' },
  { id: 'jdm', label: 'JDM' },
  { id: 'euro', label: 'Euro' },
  { id: 'supercar', label: 'Supercar' },
  { id: 'american-muscle', label: 'American Muscle' },
  { id: 'track', label: 'Track/Motorsport' },
];
```

Use single quotes for string values in the generated JS. Escape any single quotes in caption/alt text with backslash.

10. **Log summary:** `Generated gallery-images.js with {N} images`.

IMPORTANT: The categories array is hardcoded (not derived from Cloudinary data) to preserve the exact same category order and labels as v1. The `isPlaceholder` field is always `false` -- no placeholder entries.
