**A. Create `src/components/scroll-animations.css`:**
- `.section--hidden`: `opacity: 0; transform: translateY(25px); transition: opacity 0.8s ease, transform 0.8s ease;`
- `.section--visible`: `opacity: 1; transform: translateY(0);`
- Add `@media (prefers-reduced-motion: reduce)` block that sets `.section--hidden` to `opacity: 1; transform: none; transition: none;` so sections are always visible
- Add `.gallery__item--scroll-entering` keyframe animation (fadeInUp, 300ms) for the one-time gallery scroll stagger. This is separate from the existing `.gallery__item--entering` used by filter changes.

**B. Create `src/components/scroll-animations.js`:**
- Import gsap and ScrollTrigger: `import gsap from 'gsap'; import { ScrollTrigger } from 'gsap/ScrollTrigger';`
- `gsap.registerPlugin(ScrollTrigger);`
- Export `initScrollAnimations()` that calls three sub-functions:

  1. `initSectionReveals()`:
     - Check `prefers-reduced-motion` -- if reduce, return immediately
     - Select ALL `.section` elements and add `.section--hidden` class
     - Create IntersectionObserver with `threshold: 0.15`
     - On intersect: add `.section--visible`, unobserve the element
     - Observe all sections

  2. `initHeroParallax()`:
     - Use `gsap.matchMedia()` to wrap in `(prefers-reduced-motion: no-preference)` condition
     - `gsap.to('.hero__image', { yPercent: -25, ease: 'none', scrollTrigger: { trigger: '.hero', start: 'top top', end: 'bottom top', scrub: true } })`
     - This layers parallax ON TOP of the existing Ken Burns CSS animation. The Ken Burns uses `transform: scale()` via keyframes, and GSAP will add `translate()` via its own transform -- GSAP handles transform concatenation automatically.

  3. `initGalleryScrollStagger()`:
     - Check `prefers-reduced-motion` -- if reduce, return immediately
     - Get `#gallery-grid` element
     - Create IntersectionObserver with `threshold: 0.1`
     - On first intersect: select all `.gallery__item:not(.gallery__item--hidden)`, apply staggered animation delay (`i * 30ms`), add class `.gallery__item--scroll-entering`
     - After total animation duration (300 + items.length * 30)ms, remove the class and clear animation-delay styles
     - Disconnect observer (one-time only)
     - IMPORTANT: Set a flag `window.__galleryInitialAnimDone = true` after the stagger completes. This prevents collision with filter animations if user clicks a filter during entrance.

**C. Update `src/components/gallery.js`:**
- At the top of `filterGallery()`, add early return or wait if `window.__galleryInitialAnimDone` is not yet true: if the initial scroll animation hasn't completed, skip the stagger portion of filter (just hide/show without stagger) OR set `window.__galleryInitialAnimDone = true` to cancel remaining entrance animation.

**D. Update `src/main.js`:**
- Add `import './components/scroll-animations.css';` with CSS imports
- Add `import { initScrollAnimations } from './components/scroll-animations.js';`
- Call `initScrollAnimations();` after existing init calls

**E. Fix hamburger bug in `src/components/nav.js`:**
- The bug: after scrolling past the hero, opening the mobile overlay "acts weird." Root cause is likely the `window.scrollTo(0, savedScrollY)` call in `closeOverlay()` which uses smooth scrolling (from `html { scroll-behavior: smooth }`) causing a visible scroll animation instead of instant jump.
- Fix: Change `window.scrollTo(0, savedScrollY)` to `window.scrollTo({ top: savedScrollY, behavior: 'instant' })` in the `closeOverlay()` function.
- This ensures the page restores to its previous position instantly when the overlay closes, without the smooth scroll animation causing visual weirdness.
