  <action>
Create three Netlify Functions following the exact pattern from trigger-rebuild.mjs (CORS headers, OPTIONS preflight, JWT auth via context.clientContext.user).

**netlify/functions/list-posts.mjs** (GET):
- Fetch `https://api.github.com/repos/${process.env.GITHUB_OWNER}/${process.env.GITHUB_REPO}/contents/content/blog` with Authorization header `token ${process.env.GITHUB_TOKEN}`
- If 404, return `{ posts: [] }` (empty blog)
- Filter to `.md` files only
- For each file, fetch its content URL to get the full file content
- Decode Base64 content with `Buffer.from(data.content, 'base64').toString('utf-8')`
- Extract frontmatter using simple regex: `decoded.match(/^---\n([\s\S]*?)\n---/)` then parse key-value pairs (no gray-matter needed server-side)
- Also extract the markdown body (everything after the second `---`)
- Return array of objects: `{ slug, sha, title, date, cover, excerpt, status, tags, body }`
- Include `body` so the editor can load post content for editing without a separate fetch

**netlify/functions/save-post.mjs** (POST):
- Parse request body: `{ slug, content, sha, title }` where `content` is the full markdown string (frontmatter + body)
- PUT to `https://api.github.com/repos/${process.env.GITHUB_OWNER}/${process.env.GITHUB_REPO}/contents/content/blog/${slug}.md`
- Base64 encode content with `Buffer.from(content, 'utf-8').toString('base64')` (NOT btoa -- handles UTF-8 correctly)
- If `sha` is provided, include it in the request body (required for updates). Omit for new files.
- Commit message: `blog: ${sha ? 'update' : 'create'} "${title}"`
- On success, call the rebuild trigger: `await fetch(process.env.NETLIFY_BUILD_HOOK, { method: 'POST' })` directly (same logic as trigger-rebuild.mjs, inlined to avoid function-to-function calls)
- Return `{ sha: response.content.sha, message: 'Post saved' }` so the client can track the new SHA for future updates

**netlify/functions/delete-post.mjs** (POST):
- Parse request body: `{ slug, sha, title }`
- DELETE to `https://api.github.com/repos/${process.env.GITHUB_OWNER}/${process.env.GITHUB_REPO}/contents/content/blog/${slug}.md`
- Must include `sha` and commit message: `blog: delete "${title}"`
- On success, trigger rebuild (same inline approach as save-post)
- Return `{ message: 'Post deleted' }`

All three functions must:
- Return CORS headers (Access-Control-Allow-Origin: *, Access-Control-Allow-Headers: Content-Type, Authorization)
- Handle OPTIONS preflight with 204
- Check JWT: `const { user } = context.clientContext || {}; if (!user) return 401`
- Use Accept header `application/vnd.github.v3+json` for GitHub API calls
- Return proper error messages with appropriate status codes
