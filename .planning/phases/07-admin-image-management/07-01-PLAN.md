Create 4 new Netlify Functions following the EXACT boilerplate pattern from upload-image.mjs (import cloudinary, config, handler with CORS headers, OPTIONS preflight, JWT validation via context.clientContext.user).

**netlify/functions/update-image.mjs** (POST):
- Parse body: `{ public_id, caption, alt, category }` (all optional except public_id)
- Build context string in pipe-delimited format. IMPORTANT: When updating context, include ALL metadata fields (caption, alt, sort_order) -- Cloudinary's `api.update()` with context REPLACES the entire context, not merges. So first fetch the current resource to get existing values, then merge with the provided updates.
- Steps:
  1. Fetch current resource: `await cloudinary.api.resource(public_id, { context: true })`
  2. Merge existing context with updates (preserve sort_order if not provided, preserve caption/alt if not provided)
  3. Call `await cloudinary.api.update(public_id, { context: contextString })` with the merged context
  4. If `category` is provided and differs from current category (extract from public_id path), rename: `await cloudinary.uploader.rename(public_id, newPublicId, { overwrite: false })` where newPublicId is `gallery/${category}/${filename}` (filename = last segment of public_id)
  5. Return `{ success: true, public_id: finalPublicId }` (finalPublicId may differ if renamed)
- Error handling: catch and return 500 with error message. If rename fails with "already exists", return 409 Conflict.

**netlify/functions/delete-image.mjs** (POST):
- Parse body: `{ public_id }`
- Call `await cloudinary.uploader.add_tag('hidden', [public_id])`
- Return `{ success: true }`

**netlify/functions/restore-image.mjs** (POST):
- Parse body: `{ public_id }`
- Call `await cloudinary.uploader.remove_tag('hidden', [public_id])`
- Return `{ success: true }`

**netlify/functions/reorder-images.mjs** (POST):
- Parse body: `{ images: [{ public_id, sort_order }] }` -- array of objects
- Validate images array exists and is non-empty
- Loop through each image SEQUENTIALLY (not Promise.all -- avoid rate limits):
  1. Fetch current resource context: `await cloudinary.api.resource(img.public_id, { context: true })`
  2. Merge existing context with new sort_order
  3. Call `await cloudinary.api.update(img.public_id, { context: mergedContextString })`
- Return `{ success: true, updated: images.length }`
- Error handling: if any update fails, continue with remaining, collect errors, return `{ success: partial, updated: N, errors: [...] }`

For the context merge pattern used in update-image and reorder-images, extract existing context like this:

