**Modify src/components/gallery.js** to add PhotoSwipe integration:

```javascript
import PhotoSwipeLightbox from 'photoswipe/lightbox';
import PhotoSwipeDynamicCaption from 'photoswipe-dynamic-caption-plugin';
import 'photoswipe/style.css';
import 'photoswipe-dynamic-caption-plugin/photoswipe-dynamic-caption-plugin.css';
```

2. Create a module-level variable `let lightbox = null;` to hold the current PhotoSwipe instance.

3. Create `initLightbox()` function:
```javascript
function initLightbox() {
  // Destroy existing instance if any
  if (lightbox) {
    lightbox.destroy();
    lightbox = null;
  }

  lightbox = new PhotoSwipeLightbox({
    gallery: '#gallery-grid',
    // Only target visible (non-hidden) real image links (not placeholders)
    children: '.gallery__item:not(.gallery__item--hidden):not(.gallery__item--placeholder) a',
    pswpModule: () => import('photoswipe'),
    bgOpacity: 0.95,
    loop: true,
    padding: { top: 20, bottom: 40, left: 0, right: 0 },
  });

  // Dynamic captions showing "Car Name -- Category"
  const captionPlugin = new PhotoSwipeDynamicCaption(lightbox, {
    type: 'below',
    captionContent: (slide) => {
      return slide.data.element?.dataset.caption || '';
    },
  });

  lightbox.init();
}
```

4. Call `initLightbox()` at the end of `initGallery()`, after rendering items.

5. **Critical: Reinitialize on filter change.** In the `filterGallery()` function, after the filter animation completes (after the setTimeout that applies hidden/visible classes), call `initLightbox()` to reinitialize PhotoSwipe with the new set of visible items. This ensures arrow key / swipe navigation only cycles through the active category's images.

6. Hide the image counter via PhotoSwipe. Try `counterEl: false` in options first. If that doesn't work (the option name may differ), add a CSS fallback.

7. Prevent the default anchor behavior -- the `<a>` tags have `target="_blank"` as fallback, but PhotoSwipe intercepts clicks. Ensure PhotoSwipe's click handler prevents navigation. This should work by default with PhotoSwipe's gallery/children config, but verify the `<a>` elements don't navigate away.
