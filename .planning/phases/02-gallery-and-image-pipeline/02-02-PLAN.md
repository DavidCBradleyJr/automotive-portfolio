**src/components/gallery.css** -- Complete gallery styles:

1. **Gallery section override:** Override .section max-width to 1400px for the gallery. On mobile, remove side padding for full-bleed images.

2. **Filter pills (.gallery__filters):**
   - `position: sticky; top: var(--nav-height); z-index: 90;`
   - Background matches site bg (--color-bg) with bottom border or subtle shadow for separation
   - Horizontal flexbox row, centered, wrapping allowed
   - Gap: var(--space-sm)
   - Padding: var(--space-md) 0

3. **Individual pill (.gallery__pill):**
   - Pill shape: border-radius 9999px (fully rounded)
   - Inactive: transparent bg, 1px solid border using --color-border, text color --color-text-muted
   - Active (.gallery__pill--active): bg --color-accent (#7C3AED), text white, no border
   - Hover on inactive: subtle bg highlight
   - Font: var(--font-body), var(--text-sm), font-weight 500
   - Padding: var(--space-xs) var(--space-md) (compact pills)
   - Cursor pointer, transition on background/color/border 0.2s ease
   - No focus outline override -- let browser default work for a11y

4. **Masonry grid (.gallery__grid):**
   - `column-count: 3; column-gap: 20px;` at 1440px+
   - `column-count: 2; column-gap: 16px;` at 768px-1439px
   - `column-count: 1; column-gap: 0; padding: 0;` below 768px (full-bleed mobile)

5. **Gallery item (.gallery__item):**
   - `break-inside: avoid; margin-bottom: 20px;` (matches column-gap)
   - Mobile: `margin-bottom: 8px;` (thin gap between full-bleed images)
   - Overflow hidden, border-radius: 8px (desktop only, 0 on mobile for full-bleed)
   - Position relative (for potential overlay effects)

6. **Gallery image (.gallery__img):**
   - `width: 100%; height: auto; display: block;`
   - LQIP blur state: `filter: blur(20px); transform: scale(1.05);` (scale hides blur edge artifacts)
   - Loaded state (.gallery__img--loaded): `filter: blur(0); transform: scale(1);`
   - Transition: `filter 0.5s ease, transform 0.5s ease;`

7. **Placeholder items (.gallery__item--placeholder):**
   - Background gradient applied via inline style
   - Min-height: 200px (so gradient placeholders have visible area)
   - Display a subtle camera icon or "coming soon" text via ::after pseudo-element (optional, use discretion)

8. **Filter animation classes:**
   - `.gallery__item--fading`: `opacity: 0; transform: scale(0.95); transition: opacity 0.25s ease, transform 0.25s ease;`
   - `.gallery__item--hidden`: `display: none;`
   - `.gallery__item--entering`: animate in with `@keyframes galleryEnter { from { opacity: 0; transform: translateY(10px); } to { opacity: 1; transform: translateY(0); } }` duration 0.3s, ease-out, fill-mode forwards. Use animation-delay set via JS inline style for stagger.

9. **CTA wrap (.gallery__cta-wrap):**
   - `text-align: center; padding: var(--space-xl) 0;`

10. **prefers-reduced-motion:** Disable all gallery transitions/animations (blur-up, filter fade, stagger enter).

**2. Render gallery items:**
- Import `galleryImages` from `../data/gallery-images.js`.
- For each image, create a gallery item element. The HTML structure for each item:

For real images:
```html
<div class="gallery__item" data-category="jdm">

     data-caption="JDM Street Racer -- JDM"
     target="_blank">
    <img class="gallery__img"
         src="data:image/webp;base64,..."
         data-src="/images/gallery/jdm-street-racer-01.webp"
         alt="Japanese domestic market street racing car"
         width="2000"
         height="1333"
         loading="lazy" />
  </a>
</div>
```

For placeholder images:
```html
<div class="gallery__item gallery__item--placeholder" data-category="jdm"
     style="background: linear-gradient(135deg, #1a1a2e, #e94560); aspect-ratio: 3/2;">
</div>
```

- The `<a>` wrapper is needed for PhotoSwipe (Plan 03). Set `data-pswp-width` and `data-pswp-height` attributes on the anchor. Caption format: "{caption} -- {Category Label}" (e.g., "Nissan GT-R R34 -- JDM"). Use the categories array to map category id to label.
- Real images start with LQIP base64 as `src` and full URL in `data-src`.
- Placeholder items have no `<img>` or `<a>` -- just a colored gradient div with aspect-ratio.

**3. LQIP lazy loading with IntersectionObserver:**
- Create an IntersectionObserver with `rootMargin: '200px'` (preload slightly before viewport).
- Observe all `.gallery__img` elements.
- On intersection: set `img.src = img.dataset.src`, on `img.onload` add class `gallery__img--loaded`, then `unobserve`.
- This swaps the tiny base64 LQIP for the full WebP and removes the blur filter via CSS transition.

**4. Filter animation logic:**
Implement the smooth filter sequence from RESEARCH.md:

```
function filterGallery(category) {
  // 1. Update active pill
  // 2. Get all gallery items
  // 3. Phase 1: Add 'gallery__item--fading' to items that will be hidden
  // 4. After 250ms (fade duration):
  //    a. Apply 'gallery__item--hidden' to non-matching items
  //    b. Remove 'gallery__item--hidden' from matching items
  //    c. Remove 'gallery__item--fading' from all items
  // 5. Phase 2: requestAnimationFrame -> add 'gallery__item--entering' to visible items
  //    with staggered animation-delay (30ms per item via inline style)
  // 6. After animation completes (~300ms + stagger), remove 'gallery__item--entering'
  //    and clear inline animation-delay
}
```

"All" category shows everything. Other categories match `item.dataset.category === category`.

Important: After filtering, re-observe any newly visible images that haven't loaded yet (their LQIP may still be showing if they were hidden before scrolling to them).
