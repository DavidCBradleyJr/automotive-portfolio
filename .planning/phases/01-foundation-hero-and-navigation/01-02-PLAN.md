<task type="auto">
  <name>Task 1: Hero section styles with Ken Burns animation and responsive layout</name>
  <files>src/components/hero.css</files>
  <action>
Create `src/components/hero.css` with the following styles:

1. **Hero container** (`.hero`):
   - `position: relative; overflow: hidden;`
   - `height: 100vh; height: 100dvh;` (100vh fallback for older browsers, 100dvh for mobile Safari toolbar fix per RESEARCH.md Pitfall 2)
   - `display: flex; align-items: flex-end;` (positions content at bottom)
   - `width: 100%;`

2. **Hero image** (`.hero__image`):
   - `position: absolute; inset: 0;`
   - `width: 100%; height: 100%; object-fit: cover;`
   - `animation: kenburns 25s ease-in-out infinite alternate;`
   - `will-change: transform;`
   - Ken Burns keyframes: from `scale(1) translate3d(0, 0, 0)` to `scale(1.08) translate3d(-2%, -1%, 0)` -- subtle slow zoom with slight pan for cinematic feel
   - `z-index: 1;`

3. **Dark overlay** (`.hero__overlay`):
   - `position: absolute; inset: 0;`
   - `background: var(--hero-overlay);` (gradient bottom-up, dark at bottom where text is, transparent at top)
   - `z-index: 2;`
   - `pointer-events: none;`

4. **Hero content** (`.hero__content`):
   - `position: relative; z-index: 3;` (above overlay and image)
   - `padding: var(--space-xl) var(--space-lg);` (generous padding from edges)
   - `max-width: 800px;`
   - Bottom-left positioning (already handled by flex align-items: flex-end on parent)

5. **Hero name** (`.hero__name`):
   - `font-family: var(--font-display);`
   - `font-size: var(--text-hero);` (clamp 3rem to 6.25rem)
   - `font-weight: 900;`
   - `color: var(--color-white);` (pure white for maximum impact on hero)
   - `line-height: 1.1;`
   - `letter-spacing: 0.02em;`
   - `text-transform: uppercase;`
   - `margin-bottom: var(--space-sm);`

6. **Hero tagline** (`.hero__tagline`):
   - `font-family: var(--font-body);`
   - `font-size: var(--text-xl);` on mobile, `var(--text-2xl)` on desktop
   - `color: var(--color-text);` (#E0E0E0)
   - `font-weight: 300;` (light weight for elegance)
   - `margin-bottom: var(--space-lg);`

7. **Hero CTA** (`.hero__cta`):
   - Inherits `.btn .btn--primary` styles from style.css
   - Additional: `font-size: var(--text-lg);` `padding: var(--space-sm) var(--space-lg);`
   - `text-decoration: none;`

8. **Entrance animation** (CSS-driven, JS adds a trigger class):
   - `.hero__name`, `.hero__tagline`, `.hero__cta` start with `opacity: 0; transform: translateY(30px);`
   - When `.hero--loaded` class is on the hero section, transition to `opacity: 1; transform: translateY(0);`
   - Use `transition: opacity 0.8s ease, transform 0.8s ease;`
   - Stagger using transition-delay: name 0.2s, tagline 0.5s, CTA 0.8s
   - This approach avoids CSS animation (which plays immediately) in favor of a class-toggle that JS controls

9. **prefers-reduced-motion**:
   - `.hero__image { animation: none; }` (disables Ken Burns)
   - `.hero__name, .hero__tagline, .hero__cta { opacity: 1; transform: none; transition: none; }` (content visible immediately, no entrance animation)

10. **Responsive adjustments**:
    - Mobile (default): hero content padding var(--space-lg) var(--space-md), name clamp handles sizing
    - Tablet (768px+): increase content padding
    - Desktop (1024px+): hero content padding var(--space-xl) var(--space-xl), tagline size bump
  </action>
