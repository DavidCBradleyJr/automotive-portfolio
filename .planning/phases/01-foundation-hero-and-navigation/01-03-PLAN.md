<task type="auto">
  <name>Task 2: Navigation JS -- scroll-spy, frosted glass toggle, hamburger, accessibility</name>
  <files>src/components/nav.js, src/main.js</files>
  <action>
1. Create `src/components/nav.js` exporting an `initNav()` function that sets up:

   a. **Frosted glass transition** (from RESEARCH.md Pattern 4):
      - Create an IntersectionObserver watching the `.hero` section
      - When hero is NOT intersecting (user scrolled past), add `.nav--scrolled` to `.nav`
      - When hero IS intersecting (user at top), remove `.nav--scrolled`
      - Use `{ threshold: 0.1 }` so it transitions when 90% of hero is out of view

   b. **Scroll-spy** (from RESEARCH.md Pattern 3):
      - Select all `section[id]` elements and all `.nav__link` elements
      - Create an IntersectionObserver with `rootMargin: '-20% 0px -80% 0px'` (narrow detection band near top of viewport)
      - When a section enters the intersection, toggle `.nav__link--active` on the corresponding nav link (match by `href === '#' + section.id`)
      - Also update active state on overlay links if they exist

   c. **Hamburger toggle** (from RESEARCH.md Pattern 5):
      - Select `.nav__hamburger`, `.nav__overlay`, and all overlay links
      - On hamburger click: toggle `.nav__hamburger--open`, toggle `.nav__overlay--open`, toggle `document.body.classList.toggle('no-scroll')`, update `aria-expanded` attribute
      - On overlay link click: close overlay (remove all open classes, remove no-scroll, set aria-expanded false)
      - On `Escape` key: close overlay if open
      - On close: return focus to hamburger button (accessibility per RESEARCH.md Pitfall 5)

   d. **Focus trapping** (accessibility):
      - When overlay is open, trap Tab focus within the overlay (cycle through overlay links and CTA)
      - When overlay closes, restore focus to hamburger button

2. Update `src/main.js`:
   - Add: `import './components/nav.css';`
   - Add: `import { initNav } from './components/nav.js';`
   - Call `initNav();` after `initHero();`
